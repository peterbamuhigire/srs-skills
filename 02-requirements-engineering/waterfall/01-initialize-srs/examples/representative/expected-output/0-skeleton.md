# Software Requirements Specification -- Hospital Admission System

## 1. Introduction

### 1.1 Purpose

This SRS defines the functional and non-functional requirements for the
Hospital Admission System (HAS), a replacement for the legacy paper
intake flow. It traces every requirement to **BG-001** (admission-time
reduction) or **BG-002** (HIPAA compliance).

### 1.2 Scope

The system accepts patient demographics, assigns a unique medical record
number, and publishes the intake event to the downstream EMR.

### 1.3 Definitions

See `_context/glossary.md` for the authoritative glossary. All HIPAA
acronyms are spelled out on first use.
