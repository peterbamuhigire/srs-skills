# Governance And Engagement

Use this reference when the project needs a structured business analysis operating model.

## Governance Minimums

- One accountable owner for scope decisions
- One accountable owner for requirements approval
- A named escalation path for deadlocked tradeoffs
- Defined artifact review order before baselining

## Engagement Matrix

| Stakeholder Type | Preferred Mode | Typical Trigger | Output |
|---|---|---|---|
| Sponsor | Decision review | Scope, budget, go/no-go | Decision record |
| Primary users | Workshop or scenario review | Workflow discovery | Feedback log |
| Regulators/compliance | Formal review | Policy-sensitive requirements | Compliance comments |
| Operations/support | Run-through or checklist review | Readiness and transition work | Operations constraints |

## Gate Evidence

- Discovery-complete: stakeholder coverage, open questions list, assumptions list
- Design-ready: approved requirements set, process models, rules, data model
- Go/no-go: test evidence, operational readiness, unresolved-risk statement
