## 4. DoR Compliance and Enforcement

### 4.1 Responsibility

Peter Bamuhigire is responsible for ensuring every task assigned to a developer satisfies the DoR before assignment. This is not delegatable.

### 4.2 Developer Obligation

A developer who receives a task that is missing any DoR criterion must flag it to Peter Bamuhigire immediately — before beginning any implementation work. Starting work on a task that is not Ready is not acceptable.

### 4.3 Gap-Blocked Tasks

A task blocked by an unresolved critical external gap (GAP-xxx) remains in a "Blocked" state in the project task list. Peter Bamuhigire is responsible for tracking gap resolution and updating the task status when the gap is closed. Gap resolution is reviewed at every milestone kickoff.

### 4.4 Refinement

Tasks that are not yet Ready are held in a "To Refine" state. Peter Bamuhigire refines them by completing the missing criteria — typically by consulting the context files, the SRS documents, or the client. A task may not move from "To Refine" to "Ready" without Peter Bamuhigire's explicit sign-off.

### 4.5 DoR Updates

This document is versioned. Any change to the DoR criteria requires a written decision recorded in the project change log. The version number increments on every change. All developers on the project are notified of DoR changes before the updated criteria take effect.
