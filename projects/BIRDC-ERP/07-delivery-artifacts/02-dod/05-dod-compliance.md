## 4. DoD Compliance and Enforcement

### 4.1 Responsibility

Peter Bamuhigire is the DoD gatekeeper. A task may not be marked complete by any developer without Peter Bamuhigire's explicit confirmation that all applicable criteria are met.

### 4.2 DoD Checklist

Every task in the project management system carries a DoD checklist. The developer completing the task ticks each applicable criterion and confirms readiness for review. Peter Bamuhigire verifies the checklist before approving the task.

### 4.3 Waiver Prohibition

No DoD criterion may be waived to meet a deadline. If a criterion cannot be met, the task returns to in-progress status. Schedule pressure from the client does not constitute grounds for a DoD waiver.

### 4.4 Criterion Inapplicability

If a criterion is genuinely not applicable to a specific task (for example, a task that introduces no API endpoints has no RBAC criterion to satisfy), the developer marks it "N/A" with a one-sentence justification. Peter Bamuhigire must agree that the N/A designation is correct before approving.

### 4.5 DoD Updates

This document is versioned. Any change to the DoD criteria requires a written decision recorded in the project change log, with the date and the reason for the change. Both Peter Bamuhigire and the BIRDC Director must agree to any change that relaxes a criterion.
