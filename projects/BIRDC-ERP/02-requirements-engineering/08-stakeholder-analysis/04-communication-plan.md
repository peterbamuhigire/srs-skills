# Section 4: Stakeholder Communication Plan

The communication plan defines the format, frequency, owner, and audience for every structured communication channel in the BIRDC ERP programme. All communication follows the principle of minimum necessary information: each audience receives only the information relevant to their role in the programme.

---

## 4.1 Communication Register

| Communication Type | Frequency | Format | Owner | Primary Audience | Content |
|---|---|---|---|---|---|
| **Programme Steering Meeting** | Monthly | In-person or video call, 90 minutes, formal agenda | Consultant (STK-005) chairs; BIRDC Director approves agenda | STK-001 (Director), STK-002 (Finance Director), STK-003 (IT Admin), STK-005 (Consultant) | Phase progress vs. plan; risks and issues; scope change requests; phase-gate decisions; budget status |
| **Finance Director Weekly Review** | Weekly (during active phases) | In-person or video call, 60 minutes | Consultant (STK-005) | STK-002 (Finance Director), STK-005 (Consultant) | Requirements review; specification drafts for comment; open financial module questions; V&V gap review |
| **Phase Progress Report** | At completion of each phase | Written report (PDF), distributed via email | Consultant (STK-005) | STK-001, STK-002, STK-003, STK-004 (Parliament Committee) | Phase deliverables completed; milestones achieved; issues resolved; next phase plan; outstanding risks |
| **Phase Gate Review and Sign-Off** | Once per phase (7 total) | Formal meeting with sign-off document | Consultant (STK-005) facilitates | STK-001 (sign-off authority), STK-002 (financial modules), STK-003 (technical sign-off) | Phase deliverable review; acceptance criteria confirmation; go/no-go decision; next phase authorisation |
| **Parliamentary Progress Brief** | Semi-annual (aligned to parliamentary calendar) | Written brief (PDF) + oral presentation | Finance Director (STK-002); supported by Consultant | STK-004 (Parliament Budget Committee), STK-001 | System delivery progress; financial accountability narrative; value-for-money evidence; planned milestones |
| **Requirements Workshop** | Once per module group (per phase) | Workshop, 2–4 hours | Consultant (STK-005) facilitates | Relevant department heads and end users per phase (see RACI) | Requirements elicitation; business process review; edge case capture; glossary alignment |
| **UAT Session Reports** | Once per phase (delivered to sign-off authority) | Written report with test results | Consultant (STK-005) compiles; IT Admin reviews | Phase Accountable stakeholder; Finance Director for financial modules | UAT pass/fail results; open defects with severity; sign-off recommendation |
| **Agent Network Briefing** | At Phase 1 go-live + quarterly thereafter | Briefing via Sales Manager + printed materials | Sales and Marketing Manager (STK-006) | STK-015 (Field Sales Agents) | Sales Agent App update notes; remittance process changes; commission policy updates |
| **Cooperative Farmer Payment Statement** | After each payment batch | SMS confirmation (individual) + PDF statement (cooperative level) | Payroll / Accounts Payable function | STK-020 (Cooperative Farmers), STK-029 (Farmers' Unions) | Delivery quantities; quality grades; gross payable; deductions; net paid; mobile money reference |
| **EFRIS Compliance Summary** | Monthly | PDF report | Finance Director (STK-002) | STK-022 (URA) — via EFRIS portal; internal copy to STK-001, STK-002 | Total fiscal documents submitted; failed submissions resolved; FDN register |
| **PPDA Procurement Compliance Report** | Annual (or on request) | PDF formatted per PPDA requirements | Procurement Manager (STK-007) | STK-027 (PPDA); STK-001 (Director) | All procurement transactions by category; document compliance status; value summary |
| **OAG Audit Package** | On-demand (at audit initiation) | System-generated audit trail export + financial statements | Finance Director (STK-002) with IT Administrator | STK-026 (Auditor General) | GL hash chain integrity report; full transaction audit trail; financial statements; budget vs. actual |
| **Risk and Issue Register Update** | Bi-weekly | Shared document (Google Docs or SharePoint) | Consultant (STK-005) | STK-001, STK-002, STK-003 | Open risks with probability, impact, and mitigation actions; open issues with resolution owner and target date |
| **Integration Test Results** | At Phase 7 integration milestones | Technical report (PDF) | IT Administrator (STK-003) | STK-002, STK-001, STK-005 | EFRIS API test results; mobile money API test results; ZKTeco integration test results; open items |
| **Handover Documentation** | At final project close | Full documentation package (PDF + digital) | Consultant (STK-005) | STK-003 (IT Administrator primary); STK-001, STK-002 | System architecture; configuration guide; user manuals; support SOP; training materials |

## 4.2 Communication Channels

| Stakeholder Group | Primary Channel | Secondary Channel | Notes |
|---|---|---|---|
| BIRDC Director | In-person meeting + WhatsApp brief | Email | 5-minute daily brief via Executive Dashboard App (self-serve) |
| Finance Director | In-person meeting + Email | WhatsApp | Primary requirements and sign-off channel |
| IT Administrator | Email + shared project folder | In-person | Technical documentation shared via agreed folder |
| Parliament Budget Committee | Formal written report via Finance Director | In-person presentation | BIRDC Director or Finance Director presents |
| Department Heads | Email + workshop invitations | In-person meetings | Requirements workshops are primary engagement |
| Field Sales Agents | Sales Manager briefing + printed handouts | Sales Agent App in-app messages | Direct agent communication managed by Sales Manager |
| Cooperative Farmers | SMS payment notifications | Paper receipts (field) | Farmers' unions receive aggregate reports |
| External Regulators (URA, NSSF, PPDA, OAG) | Formal written submission + system-generated reports | In-person review | Finance Director or Procurement Manager leads regulatory communication |
