## How to Read This Test Plan

Each test case follows this format:

**TC-xxx** | Module | Test Description | Preconditions | Test Steps | Expected Result | Priority

Priority levels: **P1** = Critical, **P2** = High, **P3** = Medium, **P4** = Low.

Financial calculation test cases include exact-value test oracles. "PASS" means the system produces the exact value stated. Any deviation — including rounding by ±1 UGX — is a defect.

Test case identifiers are grouped by module prefix:

| Prefix | Module |
|---|---|
| TC-SAL | Sales & Distribution |
| TC-POS | Point of Sale |
| TC-INV | Inventory & Warehouse Management |
