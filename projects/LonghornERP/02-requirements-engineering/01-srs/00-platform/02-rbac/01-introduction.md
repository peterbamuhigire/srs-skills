## 1. Introduction

### 1.1 Purpose

This Software Requirements Specification (SRS) defines the functional and non-functional requirements for the User Management and Role-Based Access Control (RBAC) module of Longhorn ERP. This document governs all user lifecycle operations, role and permission management, multi-factor authentication, biometric device integration, and USSD access for low-bandwidth users. It is the authoritative reference for design, development, testing, and V&V activities related to this module.

### 1.2 Scope

The User Management and RBAC module is a Core Module of Longhorn ERP, active for every tenant and non-disableable. It governs:

- **User lifecycle management:** creation, activation, deactivation, password policy enforcement, and account lockout.
- **Role and permission management:** role definition, cloning, assignment, and the function × action permission matrix (view, create, edit, approve, delete).
- **Branch and approval-limit restrictions:** confining a user's data access and transaction authority to their assigned branches and configurable UGX thresholds.
- **Module access gating:** preventing access to functions belonging to modules the tenant has not activated.
- **Super admin impersonation:** enabling Chwezi Core Systems support staff to act on behalf of any tenant user with full audit accountability.
- **2FA and biometric authentication:** Time-based One-Time Password (TOTP) enrollment and enforcement, backup code generation, and Zkteco biometric device integration.
- **USSD access:** Africa's Talking USSD gateway for warehouse workers requiring stock lookup and Goods Receipt Note (GRN) confirmation on feature phones.

This document covers the web panel (`/public/`), the Super Admin Panel (`/public/superadmin/`), the mobile API (`/public/api/mobile/v1/`), and the USSD interface (module code: `USER_MGMT`).

### 1.3 Definitions, Acronyms, and Abbreviations

All terms are defined per IEEE Std 610.12-1990. The following terms are specific to this document; all other acronyms are defined in the project glossary (`_context/glossary.md`).

| Term | Definition |
|---|---|
| 2FA | Two-Factor Authentication. A security mechanism requiring the user to present two independent credentials before access is granted. |
| TOTP | Time-based One-Time Password. A 6-digit code generated by an authenticator application (e.g., Google Authenticator, Authy) using a shared secret and the current UTC time, valid for 30 seconds per RFC 6238. |
| Impersonation Session | A super admin session in which the operator acts with the identity of a specific tenant user. Every action taken is logged with an `impersonation_flag = TRUE` and the `impersonator_id`. |
| Permission Matrix | The two-dimensional structure mapping every system function to the set of permitted actions (view, create, edit, approve, delete) for a given role. |
| Backup Code | A single-use alphanumeric recovery code generated at 2FA enrollment, used when the user's authenticator device is unavailable. |
| GRN | Goods Receipt Note. A document confirming receipt of goods from a supplier. See project glossary. |
| RBAC | Role-Based Access Control. See project glossary. |
| USSD | Unstructured Supplementary Service Data. A GSM protocol for real-time session-based communication between a feature phone and an application server, used here via Africa's Talking. |
| Zkteco | A brand of biometric time-attendance hardware. Integration captures fingerprint or face recognition events and transmits them to the Longhorn ERP attendance and authentication API. |
