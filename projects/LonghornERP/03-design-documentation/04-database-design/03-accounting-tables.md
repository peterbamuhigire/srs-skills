# Accounting Module Tables

The Accounting and General Ledger (ACCOUNTING) module is a core module active for all tenants. The tables below support double-entry bookkeeping, multi-currency operations, tax compliance (Value Added Tax and Withholding Tax), budget management, and bank reconciliation.

## `chart_of_accounts`

Defines the hierarchical list of accounts used in the General Ledger (GL). Each tenant may customise their chart of accounts; a localisation-specific starter chart is pre-populated at onboarding.

| Column | Type | Constraints | Description |
|---|---|---|---|
| `id` | BIGINT UNSIGNED | PRIMARY KEY, AUTO_INCREMENT | Surrogate primary key. |
| `tenant_id` | BIGINT UNSIGNED | NOT NULL, FK → `tenants.id` | Tenant scope. |
| `code` | VARCHAR(20) | NOT NULL | Unique account code within the tenant (e.g., `1010`). |
| `name` | VARCHAR(255) | NOT NULL | Account name (e.g., "Cash at Bank — Stanbic UGX"). |
| `type` | ENUM('ASSET','LIABILITY','EQUITY','REVENUE','EXPENSE') | NOT NULL | Account classification per the accounting equation. |
| `parent_id` | BIGINT UNSIGNED | NULL, FK → `chart_of_accounts.id` | Parent account for hierarchical grouping; NULL for root-level accounts. |
| `is_active` | TINYINT(1) | NOT NULL, DEFAULT 1 | 1 = account is available for posting; 0 = deactivated. |

**Indexes:** PRIMARY (`id`), UNIQUE (`tenant_id`, `code`), (`tenant_id`, `type`, `is_active`), (`tenant_id`, `parent_id`).

## `account_ledgers`

The running ledger for each account. Each row represents one side of a journal line posting, recording the resulting balance after the entry.

| Column | Type | Constraints | Description |
|---|---|---|---|
| `id` | BIGINT UNSIGNED | PRIMARY KEY, AUTO_INCREMENT | Surrogate primary key. |
| `tenant_id` | BIGINT UNSIGNED | NOT NULL, FK → `tenants.id` | Tenant scope. |
| `journal_line_id` | BIGINT UNSIGNED | NOT NULL, FK → `gl_journal_lines.id` | The journal line that produced this ledger entry. |
| `account_id` | BIGINT UNSIGNED | NOT NULL, FK → `chart_of_accounts.id` | The account being posted to. |
| `debit` | DECIMAL(18,4) | NOT NULL, DEFAULT 0 | Debit amount in the account's base currency. |
| `credit` | DECIMAL(18,4) | NOT NULL, DEFAULT 0 | Credit amount in the account's base currency. |
| `balance` | DECIMAL(18,4) | NOT NULL | Running balance of the account after this entry. |
| `posting_date` | DATE | NOT NULL | Business date of the posting, which may differ from the transaction creation date. |

**Indexes:** PRIMARY (`id`), (`tenant_id`, `account_id`, `posting_date`), (`tenant_id`, `journal_line_id`).

---

## `gl_journal_entries`

The header record for each journal entry. A journal entry groups one or more journal lines that together satisfy the double-entry constraint (total debits = total credits).

| Column | Type | Constraints | Description |
|---|---|---|---|
| `id` | BIGINT UNSIGNED | PRIMARY KEY, AUTO_INCREMENT | Surrogate primary key. |
| `tenant_id` | BIGINT UNSIGNED | NOT NULL, FK → `tenants.id` | Tenant scope. |
| `entry_number` | VARCHAR(30) | NOT NULL | System-generated sequential reference (e.g., `JE-2026-000001`); generated by `sp_generate_entry_number`. |
| `entry_date` | DATE | NOT NULL | Business date of the journal entry. |
| `description` | TEXT | NULL | Narrative description of the entry purpose. |
| `status` | ENUM('draft','posted','reversed') | NOT NULL, DEFAULT 'draft' | Lifecycle state; only posted entries affect account balances. |
| `created_by` | BIGINT UNSIGNED | NOT NULL, FK → `users.id` | User who created the entry. |

**Indexes:** PRIMARY (`id`), UNIQUE (`tenant_id`, `entry_number`), (`tenant_id`, `status`, `entry_date`).

---

## `gl_journal_lines`

Individual debit or credit lines within a journal entry. Each line references one account and one cost centre.

| Column | Type | Constraints | Description |
|---|---|---|---|
| `id` | BIGINT UNSIGNED | PRIMARY KEY, AUTO_INCREMENT | Surrogate primary key. |
| `tenant_id` | BIGINT UNSIGNED | NOT NULL, FK → `tenants.id` | Tenant scope. |
| `journal_id` | BIGINT UNSIGNED | NOT NULL, FK → `gl_journal_entries.id` | Parent journal entry. |
| `account_id` | BIGINT UNSIGNED | NOT NULL, FK → `chart_of_accounts.id` | Account to debit or credit. |
| `debit` | DECIMAL(18,4) | NOT NULL, DEFAULT 0 | Debit amount; 0 if this line is a credit. |
| `credit` | DECIMAL(18,4) | NOT NULL, DEFAULT 0 | Credit amount; 0 if this line is a debit. |
| `cost_centre_id` | BIGINT UNSIGNED | NULL | Optional cost centre reference for management accounting segmentation. |

**Indexes:** PRIMARY (`id`), (`tenant_id`, `journal_id`), (`tenant_id`, `account_id`).

## `account_mappings`

Configures which GL accounts to debit and credit for each business event type (e.g., sales invoice posting, purchase invoice posting). Supports 3-level resolution: item-level overrides category-level, which overrides the default. The stored procedure `sp_get_account_mapping` implements this resolution.

| Column | Type | Constraints | Description |
|---|---|---|---|
| `id` | BIGINT UNSIGNED | PRIMARY KEY, AUTO_INCREMENT | Surrogate primary key. |
| `tenant_id` | BIGINT UNSIGNED | NOT NULL, FK → `tenants.id` | Tenant scope. |
| `event_type` | VARCHAR(50) | NOT NULL | Business event code (e.g., `SALES_INVOICE`, `PURCHASE_INVOICE`, `STOCK_ISSUE`). |
| `item_id` | BIGINT UNSIGNED | NULL, FK → `stock_items.id` | Item-level mapping; NULL if this rule applies at category or default level. |
| `category_id` | BIGINT UNSIGNED | NULL, FK → `item_categories.id` | Category-level mapping; NULL if item-level or default. |
| `debit_account_id` | BIGINT UNSIGNED | NOT NULL, FK → `chart_of_accounts.id` | Account to debit for this event. |
| `credit_account_id` | BIGINT UNSIGNED | NOT NULL, FK → `chart_of_accounts.id` | Account to credit for this event. |
| `level` | ENUM('ITEM','CATEGORY','DEFAULT') | NOT NULL | Explicit resolution level; used by `sp_get_account_mapping` to order lookups. |

**Indexes:** PRIMARY (`id`), (`tenant_id`, `event_type`, `level`), (`tenant_id`, `event_type`, `item_id`), (`tenant_id`, `event_type`, `category_id`).

## `tax_codes`

Defines Value Added Tax (VAT) and Withholding Tax (WHT) codes applicable to the tenant's jurisdiction.

| Column | Type | Constraints | Description |
|---|---|---|---|
| `id` | BIGINT UNSIGNED | PRIMARY KEY, AUTO_INCREMENT | Surrogate primary key. |
| `tenant_id` | BIGINT UNSIGNED | NOT NULL, FK → `tenants.id` | Tenant scope. |
| `code` | VARCHAR(20) | NOT NULL | Short code (e.g., `VAT18`, `WHT6`). |
| `name` | VARCHAR(100) | NOT NULL | Descriptive name (e.g., "VAT 18%", "WHT on Services 6%"). |
| `rate_percent` | DECIMAL(7,4) | NOT NULL | Tax rate as a percentage (e.g., 18.0000 for 18%). |
| `type` | ENUM('VAT','WHT') | NOT NULL | Tax category. |

**Indexes:** PRIMARY (`id`), UNIQUE (`tenant_id`, `code`).

---

## `bank_accounts`

Records the bank accounts operated by the tenant, linked to corresponding GL accounts for automated reconciliation.

| Column | Type | Constraints | Description |
|---|---|---|---|
| `id` | BIGINT UNSIGNED | PRIMARY KEY, AUTO_INCREMENT | Surrogate primary key. |
| `tenant_id` | BIGINT UNSIGNED | NOT NULL, FK → `tenants.id` | Tenant scope. |
| `account_name` | VARCHAR(255) | NOT NULL | Descriptive name for the bank account. |
| `bank_name` | VARCHAR(255) | NOT NULL | Name of the financial institution. |
| `account_number` | VARCHAR(50) | NOT NULL | Bank account number; stored as plain text, not a sensitive credential. |
| `currency_code` | CHAR(3) | NOT NULL, FK → `currencies.code` | ISO 4217 currency code for this account. |
| `gl_account_id` | BIGINT UNSIGNED | NOT NULL, FK → `chart_of_accounts.id` | The GL account that mirrors this bank balance. |

**Indexes:** PRIMARY (`id`), (`tenant_id`, `gl_account_id`).

---

## `currencies`

Lookup table for all supported ISO 4217 currency codes. Not tenant-scoped; managed by the platform super-admin.

| Column | Type | Constraints | Description |
|---|---|---|---|
| `id` | BIGINT UNSIGNED | PRIMARY KEY, AUTO_INCREMENT | Surrogate primary key. |
| `code` | CHAR(3) | NOT NULL, UNIQUE | ISO 4217 three-letter currency code (e.g., `UGX`, `USD`, `KES`). |
| `name` | VARCHAR(100) | NOT NULL | Full currency name (e.g., "Ugandan Shilling"). |
| `symbol` | VARCHAR(10) | NOT NULL | Display symbol (e.g., "UGX", "$", "KSh"). |
| `decimal_places` | TINYINT UNSIGNED | NOT NULL, DEFAULT 2 | Number of decimal places for display rounding. |

**Indexes:** PRIMARY (`id`), UNIQUE (`code`).

---

## `exchange_rates`

Daily exchange rates used for multi-currency invoice conversion and GL reporting. The application will use the rate with the closest `effective_date` not exceeding the transaction date.

| Column | Type | Constraints | Description |
|---|---|---|---|
| `id` | BIGINT UNSIGNED | PRIMARY KEY, AUTO_INCREMENT | Surrogate primary key. |
| `from_currency` | CHAR(3) | NOT NULL, FK → `currencies.code` | Source currency. |
| `to_currency` | CHAR(3) | NOT NULL, FK → `currencies.code` | Target currency. |
| `rate` | DECIMAL(18,8) | NOT NULL | Exchange rate: 1 unit of `from_currency` expressed in `to_currency`. |
| `effective_date` | DATE | NOT NULL | Date from which this rate is valid. |

**Indexes:** PRIMARY (`id`), (`from_currency`, `to_currency`, `effective_date`).

---

## `budgets`

Stores monthly budget amounts per account per cost centre. Used by the budget-vs-actual reporting view.

| Column | Type | Constraints | Description |
|---|---|---|---|
| `id` | BIGINT UNSIGNED | PRIMARY KEY, AUTO_INCREMENT | Surrogate primary key. |
| `tenant_id` | BIGINT UNSIGNED | NOT NULL, FK → `tenants.id` | Tenant scope. |
| `account_id` | BIGINT UNSIGNED | NOT NULL, FK → `chart_of_accounts.id` | The GL account being budgeted. |
| `cost_centre_id` | BIGINT UNSIGNED | NULL | Optional cost centre for segmented budgeting. |
| `period_year` | SMALLINT UNSIGNED | NOT NULL | Four-digit budget year (e.g., 2026). |
| `period_month` | TINYINT UNSIGNED | NOT NULL | Budget month 1–12. |
| `amount` | DECIMAL(18,4) | NOT NULL | Budgeted amount in the tenant's base currency. |

**Indexes:** PRIMARY (`id`), UNIQUE (`tenant_id`, `account_id`, `cost_centre_id`, `period_year`, `period_month`), (`tenant_id`, `period_year`, `period_month`).
