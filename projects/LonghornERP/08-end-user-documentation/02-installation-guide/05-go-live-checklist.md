# Step 5: Go-Live Checklist

Complete every item on this checklist before processing your first real transaction in Longhorn ERP. Processing transactions before setup is complete can result in incorrect tax submissions, misposted opening balances, or incomplete audit trails that are difficult to correct.

Work through the list from top to bottom. Tick each item only after you have verified it — not just configured it.

---

## Organisation Setup

- [ ] Company profile complete: legal name, physical address, and TIN/VAT number entered and correct.
- [ ] Company logo uploaded and displays correctly on invoice preview.
- [ ] Localisation confirmed: country, primary currency, and VAT/tax rate match your registration.
- [ ] PAYE bands and NSSF rates confirmed for the current financial year (HR module only).

## Financial Setup

- [ ] Chart of accounts loaded using the correct COA template for your jurisdiction.
- [ ] Opening balances entered for all balance sheet accounts as at your cutover date.
- [ ] Opening balance debit/credit difference equals zero.
- [ ] At least one active fiscal year configured in **Accounting > Fiscal Years**.

## Organisation Structure

- [ ] At least 1 branch created with a complete address.
- [ ] At least 1 warehouse created and linked to the branch.

## Users and Access

- [ ] All user accounts created with correct names and email addresses.
- [ ] Each user assigned an appropriate role.
- [ ] All users have accepted their invitation and logged in at least once.
- [ ] No users are assigned System Administrator access unless that level of access is intentional.

## Modules

- [ ] All required modules activated under **Settings > Modules**.
- [ ] Module-specific settings reviewed (e.g., invoice numbering series, stock valuation method, payroll period).

## Integrations

- [ ] URA EFRIS connection tested and returns a green status (Uganda VAT-registered businesses only).
- [ ] MTN MoMo test transaction reconciled successfully (if applicable).
- [ ] Airtel Money test transaction reconciled successfully (if applicable).
- [ ] Email delivery tested: test email received in external inbox.

## End-to-End Transaction Tests

- [ ] Test sales invoice created, posted, and printed successfully.
- [ ] Test invoice submitted to EFRIS and fiscal receipt returned (Uganda only).
- [ ] Test purchase order created and goods receipt posted successfully.
- [ ] Test payroll run completed and payslip generated (HR module only).

## Data Migration (if applicable)

- [ ] Customer records imported and validated.
- [ ] Supplier records imported and validated.
- [ ] Item catalogue imported and validated.
- [ ] Employee records imported and validated (HR module only).

## Support and Backup

- [ ] Backup configuration verified with Chwezi Core support.
- [ ] At least one staff member has been trained on daily operations for each active module.

---

## Ready to Go Live

Once all items above are ticked, your system is ready for production use. Inform your team of the go-live date and begin processing live transactions.
