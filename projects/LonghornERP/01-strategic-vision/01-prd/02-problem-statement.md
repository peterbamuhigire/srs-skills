# Problem Statement: The ERP Gap in East Africa

## The Market Failure

Mid-sized organisations in Uganda and East Africa require the same operational capabilities as enterprises — financial control, inventory tracking, procurement governance, payroll compliance, and business intelligence. The ERP products that deliver these capabilities are priced for multinational budgets, require expensive implementation projects, and are designed for markets with mature IT departments and formal training infrastructure. The result is that most mid-sized East African organisations operate on disconnected spreadsheets, entry-level accounting software, or partially deployed ERP systems they cannot fully use.

## Competitive Cost Comparison (30-User Organisation, Uganda)

| Competitor | Cost Estimate | Principal Barrier |
|---|---|---|
| SAP Business One | ~UGX 145M licence + UGX 110M–375M implementation | Capital expenditure beyond reach of any SME |
| Oracle NetSuite | ~UGX 14.9M/month | 6–100× more expensive than Longhorn ERP |
| MS Dynamics 365 BC | ~UGX 7.9M–11.3M/month + implementation | 3–45× more expensive; requires partner deployment |
| Odoo Enterprise | ~UGX 4.3M/month + UGX 75M–300M implementation | 2–17× more expensive; implementation cost often exceeds licence |
| ERPNext | Free licence + UGX 56M–188M to implement | Zero licence cost, but implementation cost excludes the market |
| **Longhorn ERP** | **UGX 250,000–2,500,000/month for the whole organisation** | No implementation fee; no per-seat fees; no consultant required |

*USD equivalents at ~UGX 3,750/$1.*

## The Core Problem

The barrier is not only price. The deeper problem is complexity: every international ERP listed above requires an implementation consultant to configure it, a trainer to teach staff to use it, and ongoing vendor support to change it. For a Ugandan organisation with 20–60 employees, this dependency model is economically and operationally incompatible.
