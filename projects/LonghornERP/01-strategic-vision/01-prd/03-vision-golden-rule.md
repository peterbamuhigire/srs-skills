# Vision, Mission, and the Golden Rule

## Vision Statement

To be the default ERP platform for mid-sized organisations across Africa — displacing spreadsheets and expensive international ERPs with a system that any employee can use on their first day, in any country, in any sector.

## Mission

Chwezi Core Systems will build, operate, and continuously improve Longhorn ERP as a fully hosted, Africa-native SaaS platform that delivers enterprise operational capability at a price and complexity level accessible to organisations with 5–500 employees.

## The Golden Rule

*If a user would need a manual or a consultant to complete a task, the interface is redesigned until they do not.*

### Elaboration

The Golden Rule is a design constraint, not a UX aspiration. It has the following operational implications:

- Every user-facing workflow must be completable without reading documentation, watching a tutorial, or requesting vendor support.
- Navigation must be discoverable: a user who has never seen the system must be able to locate any core function within 3 clicks from the dashboard.
- Error messages must state what went wrong and what the user must do to correct it — not generic codes or "an error occurred."
- Form fields must carry inline labels, placeholder examples, and validation feedback that eliminates ambiguity about required format.
- The system shall achieve a ≥ 85% unassisted task success rate on the 5 benchmark tasks defined in **Section 8** of this document.

### Compliance Measurement

The Golden Rule is evaluated through moderated usability testing. A participant who has never used the system is given a task card with no instructions. Success is defined as task completion without requesting help and without navigating to a dead end that required backtracking more than once. Failure triggers a mandatory interface redesign cycle before the feature is released.

## 5-Year Market Expansion Vision

| Phase | Market | Timeline | MRR Target (USD equiv.) |
|---|---|---|---|
| Phase 1 | Uganda (primary market) | Launch year | ~$15K MRR (UGX 56M) |
| Phase 2 | East Africa: Kenya, Tanzania, Rwanda | 24 months post-launch | ~$50K MRR (UGX 188M) |
| Phase 3 | Francophone Africa: DRC, Cameroon, Côte d'Ivoire, Senegal, Mali, Burkina Faso; South Sudan; Burundi | 36 months post-launch | ~$120K MRR (UGX 450M) |
| Phase 4 | Enterprise and global — any country addable via localisation profile with no code changes | 48–60 months post-launch | $300K+ MRR (UGX 1.1B+) |

The Phase 4 target establishes Longhorn ERP as globally configurable. Any new country market is added by authoring a localisation profile — defining currency, tax rules, statutory integrations, chart of accounts starter, and address format — without modifying application code.
