# EMIS Data Dictionary: Uganda MoES Education Management Information System

> **Source documents:** MoES EMIS Primary School User Manual (April 2023), MoES EMIS Secondary School User Manual (April 2023), Learner Registration Form (2023), Staff Registration Form (2024), International Learner Registration Form.
>
> **Purpose:** Documents the exact data structure that AcademiaPro must export to comply with the Uganda MoES EMIS system. Closes `[CONTEXT-GAP: FR-EMIS-001]`.

## 1. EMIS Portal Overview

- **Web portal:** `https://emis.go.ug/`
- **System operator:** Ministry of Education and Sports (MoES), managed via SMS ONE (U) LTD as technical consultant
- **EMIS Number:** Unique institution identifier assigned by MoES; every school must have one before accessing the portal
- **LIN (Learner Identification Number):** Auto-generated by the EMIS system upon learner registration; persists across schools and transitions
- **NIN verification:** The portal validates National Identification Numbers (NIN) against the NIRA (National Identification and Registration Authority) database in real time; auto-fills biographical fields on successful match
- **Bulk upload:** Excel workbook templates (`.xlsx`) are available for learners and staff; separate templates exist for Ugandan, Foreign, and Refugee categories

## 7. Secondary School Specific: UNEB Integration

- Secondary schools register learners using the **PLE/UCE Index Number** combined with `Exam Year`
- The EMIS portal fetches the learner's UNEB details from the UNEB database for verification before completing registration
- Subject combination capture is required for S5 and S6 students
- The `UNEB Centre Number` is stored in institution particulars and links the school to UNEB records

## 8. Data Upload Methods

| Method | Scope | Notes |
|---|---|---|
| Web form | Single record | Manual entry via `https://emis.go.ug/` portal |
| Excel workbook bulk upload | Multiple records | Separate `.xlsx` templates for learners, teaching staff, and non-teaching staff; further split by Ugandan, Foreign, and Refugee categories |

## 10. UNEB Candidate Registration Export

For FR-EXM-008: The EMIS system receives UNEB candidate data via `Index Number` + `Exam Year`.

### 10.1 Required Export Fields

| Field | Source | Notes |
|---|---|---|
| `UNEB Centre Number` | `schools.uneb_centre_number` | From institution particulars |
| `Index Number` | `students.uneb_index_number` | Assigned by UNEB, entered manually in AcademiaPro |
| `Exam Year` | `exam_registrations.exam_year` | Year of examination |
| `First Name` | `students.first_name` | |
| `Surname` | `students.last_name` | |
| `Other Names` | `students.middle_name` | |
| `Gender` | `students.gender` | Map to Male/Female |
| `Subject Entries` | `student_subjects` | Joined subject list per candidate |

### 10.2 Implementation Notes

- The exact UNEB registration file format is not published in the EMIS user manuals
- The EMIS portal interfaces with the UNEB database server-side; schools do not upload candidate files directly to UNEB through EMIS
- AcademiaPro shall support:
  1. Manual entry of Index Numbers received from UNEB
  2. Export of candidate lists in Excel format matching UNEB's expected columns for centre-level registration
