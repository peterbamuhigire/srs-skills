# Section 2: Overall Description

## 2.1 Product Perspective

The Restaurant/Bar module (F-011) is an add-on layer within the Maduuka multi-tenant Software as a Service (SaaS) platform. It does not operate independently; it requires and extends the following Phase 1 modules:

- **F-001 (Point of Sale):** F-011 reuses the POS payment processing engine (FR-POS-011 through FR-POS-016) for bill settlement and bar tab closure. All payment methods available in F-001 (cash, MTN MoMo, Airtel Money, credit) are available in F-011 without duplication of payment logic.
- **F-002 (Inventory and Stock Management):** F-011 introduces bill-of-materials records that instruct the inventory engine to deduct raw material SKUs on KOT send, rather than finished-good SKUs at point of sale. Stock movement records generated by KOT send are of type `kot_deduction` and are immutable per BR-004.

The module does not introduce a separate data store. All F-011 data (tables, KOTs, bar tabs, BOMs, floor plans) resides within the existing Maduuka tenant database, scoped by `franchise_id` per BR-001.

### 2.1.1 Phase 3 Integration Point

F-011 includes a defined interface point for Phase 3 (F-013 Hotel/Accommodation): the ability to post F&B charges to a hotel room account. This interface is *not* implemented in Phase 2a. The data model for F-011 bills will include a nullable `room_account_id` field, reserved for the Phase 3 integration. All Phase 2a bill settlement flows treat this field as null.

## 2.2 Prerequisite Conditions

The Restaurant/Bar add-on module shall only be activatable by a tenant that meets all of the following conditions:

1. The tenant has an active Maduuka Phase 1 subscription.
2. The F-001 (Point of Sale) module is active on the tenant's account.
3. The F-002 (Inventory and Stock Management) module is active on the tenant's account.
4. The tenant has purchased the F-011 add-on (UGX 30,000/month surcharge).

Tenants who deactivate F-001 or F-002 while F-011 is active shall have F-011 suspended automatically until the prerequisite modules are restored.

## 2.3 User Classes

| User Class | Description | Primary F-011 Interactions |
| --- | --- | --- |
| Business Owner | Full access to all configuration, reporting, and management functions | Floor plan designer, BOM setup, charge configuration, all reports |
| Branch Manager | All operational functions except subscription management | Table setup, area management, waiter assignment configuration, all reports |
| Waiter/Server | Table-level order entry and KOT dispatch | Table selection, order entry, KOT send, bill request |
| Kitchen Staff | KDS view and KOT status update | View KDS, mark items done, mark KOT complete |
| Cashier/POS Operator | Bill generation and payment collection | Generate bill from KOTs, apply charges, process payment, table release |
| Bar Staff | Bar tab management and drink order entry | Open tab, add rounds, transfer tab, settle tab |

Kitchen Staff access to the KDS does not require per-session authentication after the initial device login, as the KDS is a shared display screen (see Section 5.2).

## 2.5 Uganda F&B Market Context

The Uganda food and beverage sector presents two primary operator archetypes that F-011 must serve:

- **Standalone restaurants and bars:** Owner-operated or franchised dining establishments, ranging from single-room eateries to multi-area restaurant complexes. These operators require table management, KOT-based kitchen workflow, and split billing.
- **Hotel F&B outlets:** Restaurants, bars, and room-service operations within hotels, lodges, and guesthouses. These outlets share the same operational workflow as standalone F&B but additionally require the ability to post charges to a guest room account (Phase 3 scope, F-013).

Both archetypes operate in a predominantly cash-and-mobile-money economy. The system must support MTN MoMo and Airtel Money bill settlement with the same reliability as cash. Mixed-payment bills (e.g., partial MoMo + cash) are a frequent pattern.

Service charge is a common business practice in Ugandan restaurants and is typically a fixed percentage (10% is the market norm, though configurable). Cover charge is used by higher-end establishments. The system treats both as configurable per tenant.

VAT treatment of F&B items is subject to Uganda Revenue Authority guidance for EFRIS integration. See [CONTEXT-GAP: GAP-010 — confirm Uganda VAT treatment of restaurant/bar items before EFRIS integration is specified].
