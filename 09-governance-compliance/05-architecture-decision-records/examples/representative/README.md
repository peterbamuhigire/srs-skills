# Worked example -- architecture-decision-records

This example shows how the `05-architecture-decision-records` skill
turns a short context note into a Michael Nygard-format ADR with the
four canonical sections (Status, Context, Decision, Consequences) plus
an explicit Alternatives-rejected list.

The output is the exact file pattern the Phase 03 gate expects in
`03-design-documentation/adr/ADR-NNN-*.md`.
