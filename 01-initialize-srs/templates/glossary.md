# Glossary of Key Terms

<!-- Expert Guidance: Populate with US ISO/IEC 15504-1 terms, maintain industrial tone, and leave space for project-specific acronyms. -->

| Term | Definition |
|------|------------|
| Baseline | An approved version of a work product that can be changed only through formal change control procedures. |
| Process Capability | Capability of a process to achieve defined performance with specified levels of confidence per ISO/IEC 15504-1. |
| Assessment Model | Structured framework for process assessment (e.g., the ISO/IEC 15504 process dimension and capability levels). |
| Verification | Evaluation of a system or component to determine whether the products of a given development phase satisfy the imposed conditions. |
| Validation | Evaluation during or at the end of the development process to determine whether it satisfies specified requirements. |
| Traceability | Ability to describe and follow the life of a requirement, both forward and backward, throughout the project lifecycle. |
| Risk | Potential event affecting objectives; described with probability, impact, and mitigations. |
