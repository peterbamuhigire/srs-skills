# Incident Report Template

Based on BS ISO/IEC/IEEE 29119-3 Section 7.5.

- **Incident ID:** IR-{YYYY}{NNN}
- **Date/Time Reported:** [timestamp]
- **Reporter:** [role/name]
- **Severity:** Critical | Major | Minor | Trivial
- **Description:** What happened, observed behavior.
- **System State / Preconditions:** Exact system configuration or failure mode.
- **Test Case ID(s):** TC-xxx linked by traceability matrix.
- **Initial Classification:** Functional defect | Environment | Test data | Script | Security.
- **Immediate Action:** Retry, rollback, stop testing, barrier triggered.
- **Status:** Open | In Progress | Resolved | Verified | Closed
- **Validator:** Tester or automation identity
- **Resolution Summary:** Root cause, fix, regression verify results.

Use this template whenever the Test Report logs an incident per 29119-3 §7.5 requirements.
