# Worked example -- test-plan

This example shows how the `02-test-plan` skill turns a small Functional
Requirements extract into an ISO/IEC/IEEE 29119-3-compatible test plan
entry with front-matter (`phase`, `inputs`, `expected_results`,
`requirement_trace`) and stimulus-response test cases.

The front matter keys are what the engine's Phase 05 gate inspects, so
the file is both a human deliverable and a machine-verifiable artefact.
